Obtener listado de personas

Obtiene un listado de personas, o una persona en particular, para luego procesar el informe correspondiente.

GET https://www.inteligente.com.ar/api/informes

Parámetros
buscar string Este parámetro es requerido y define los criterios de búqueda y filtrado.
+ Ejemplo: "{busqueda}".
buscar_tipo string Este parámetro es opcional y define el tipo a filtrar.
+ Ejemplo: "F".
- Valores: "F" para Física, "J" para Jurídica.
buscar_sexo string Este parámetro es opcional y define el sexo a filtrar.
+ Ejemplo: "M".
- Valores: "M" para Masculino, "F" para Femenino.
buscar_provincia string Este parámetro es opcional y define la provincia a filtrar.
+ Ejemplo: "CORDOBA".
- Valores: "CORDOBA" para Córdoba.
buscar_historico string Este parámetro es opcional y define si se deben buscar los registros de informes solicitados con anterioridad.
+ Ejemplo: "I".
- Valores: "I" para Informes, "C" para Cantidad.
pagina int Este parámetro es opcional y se utiliza para indicar el número de página a consultar.
+ Ejemplo: "{busqueda_pagina}".
cURL
curl --user '{username}:{password}' -X 'GET' -H 'Content-Type: application/json' 'https://www.inteligente.com.ar/api/informes?buscar='
PHP
$informe = json_decode(file_get_contents('https://www.inteligente.com.ar/api/informes?username={username}&password={password}&buscar='));

Solicitar informe

Genera un nuevo informe para la persona seleccionada.

GET https://www.inteligente.com.ar/api/informes/solicitar/:cuit

Parámetros por URL
cuit int Este parámetro es requerido y define el CUIT de la persona.
+ Ejemplo: "{cuit}".
Parámetros
procesar_forzado string Este parámetro es opcional y fuerza la generación de un informe que ya ha sido procesado previamente.
+ Ejemplo: "1".
- Valores: "0" para No, "1" para Si.
procesar_secciones string Este parámetro es opcional y define las secciones a procesar.
+ Ejemplo: "CJ_IN".
+ Ejemplo: "CJ_IN,IC_PA".
- Valores:
. CJ_IN: Identidad (offline)
. CJ_HO: Personas con nombres similares (offline)
. SF_UI: BCRA - Deudores - Último período en CD (offline)
. SF_DH: BCRA - Historial (offline)
. SF_CD: BCRA - Cheques rechazados (offline)
. SF_EE: BCRA - Deudores morosos de ex-entidades financieras (offline)
. IA_IP: Consultas a sitios públicos - AFIP - Controles (online)
. IA_AO: AFIP - Constancia de inscripción (offline)
. IA_PB: PAMI - Beneficios (online)
. OI_SC: Dirección de aduanas (offline)
. OI_PE: Proveedores del Estado (offline)
. VL_HL: Historia laboral (offline)
. VL_AC: ANSES - Obras sociales (offline)
. JU_CB: Boletines oficiales y mesas de entrada (offline)
. JU_QU: Concursos y quiebras (offline)
. JU_AA: Facturas apócrifas en tiempo real (online)
. JU_FA: Facturas apócrifas (offline)
. JU_SO: Oficios AFIP y SOJ (offline)
. JU_SD: Posibles juicios y edictos por documento (offline)
. JU_SN: Posibles juicios y edictos por nombre (offline)
. IC_DF: Domicilio fiscal en tiempo real (online)
. IC_OD: Domicilios alternativos (offline)
. IC_PR: Relaciones personas (offline)
. IC_PE: Relaciones comerciales (offline)
. IC_DO: Teléfonos en tiempo real por el último domicilio (online)
. IC_TC: Teléfonos por CUIT o CUIL (offline)
. IC_TE: Teléfonos por domicilio (offline)
. IC_PD: Personas con domicilios coincidentes (offline)
. IC_TF: Teléfonos por domicilio fiscal (offline)
. IC_PA: Teléfonos por nombre (offline)
. RI_RC: Registro de consultas generadas por clientes (offline)
.
procesar_secciones_grupos string Este parámetro es opcional y define el grupo de secciones a procesar.
+ Ejemplo: "CJ".
+ Ejemplo: "CJ,IC".
- Valores:
. CJ: Información de identidad
. IC: Información complementaria
. IA: Información de actividad
. VL: Información laboral
. SF: Información financiera
. JU: Información judicial
. OI: Información aduanera
. RI: Registros
.
formato string Este parámetro es opcional y define el tipo de formato.
+ Ejemplo: "json".
- Valores: "json", "html", "xml".
cURL
curl --user '{username}:{password}' -X 'GET' -H 'Content-Type: application/json' 'https://www.inteligente.com.ar/api/informes/solicitar/:cuit'
PHP
$informe = json_decode(file_get_contents('https://www.inteligente.com.ar/api/informes/solicitar/:cuit?username={username}&password={password}'));

Obtener captcha de una consulta en línea

Inicia la solicitud de una consulta en línea, dicha solicitud devuelve un captcha para ser utilizado en la confirmación de la misma.

GET https://www.inteligente.com.ar/api/informes/modulos/consultas/obtener/:cuit/:informe/:consulta

Parámetros por URL
cuit int Este parámetro es requerido y define el CUIT de la persona.
+ Ejemplo: "{cuit}".
informe int Este parámetro es requerido y define el código de informe.
+ Ejemplo: "{informe}".
consulta string Este parámetro es requerido y define la configuración de las secciones a incluir en el informe.
+ Ejemplo: "bcra".
- Valores:
. bcra: BCRA deudores y cheques
. afip_controles: AFIP controles
. afip: AFIP constancia de inscripción
. afip_misaportes: AFIP mis aportes
. anses: ANSES aportes obras sociales
. anses_certificacion: ANSES actividades
. anses_cobro: ANSES lugar de cobro
. sssalud_serviciodomestico: SSSALUD servicio doméstico
.
cURL
curl --user '{username}:{password}' -X 'GET' -H 'Content-Type: application/json' 'https://www.inteligente.com.ar/api/informes/modulos/consultas/obtener/:cuit/:informe/:consulta'
PHP
$informe = json_decode(file_get_contents('https://www.inteligente.com.ar/api/informes/modulos/consultas/obtener/:cuit/:informe/:consulta?username={username}&password={password}'));

Procesar una consulta en línea

Finaliza la solicitud de una consulta en línea utilizando opcionalmente el captcha recibido en la solicitud previa.

GET, POST https://www.inteligente.com.ar/api/informes/modulos/consultas/procesar/:cuit/:informe/:consulta

Parámetros por URL
cuit int Este parámetro es requerido y define el CUIT de la persona.
+ Ejemplo: "{cuit}".
informe int Este parámetro es requerido y define el código de informe.
+ Ejemplo: "{informe}".
consulta string Este parámetro es requerido y define la configuración de las secciones a incluir en el informe.
+ Ejemplo: "bcra".
- Valores:
. bcra: BCRA deudores y cheques
. afip_controles: AFIP controles
. afip: AFIP constancia de inscripción
. afip_misaportes: AFIP mis aportes
. anses: ANSES aportes obras sociales
. anses_certificacion: ANSES actividades
. anses_cobro: ANSES lugar de cobro
. sssalud_serviciodomestico: SSSALUD servicio doméstico
.
Parámetros
datos string Este parámetro es requerido y define el captcha obtenidos previamente.
valor string Este parámetro es requerido y define el código del captcha a validar.
cURL
curl --user '{username}:{password}' -X 'POST' -H 'Content-Type: application/json' 'https://www.inteligente.com.ar/api/informes/modulos/consultas/procesar/:cuit/:informe/:consulta' -d '{"datos":"","valor":""}'

Consultar sesión

Retorna las propiedades, permisos y atributos de la cuenta en uso.

POST, PUT, GET https://www.inteligente.com.ar/api/usuarios/sesion

Parámetros
action string Este parámetro es requerido y define la acción a realizar.
+ Ejemplo: "login".
- Valores: "login", "logout".
username string Este parámetro es opcional y define el código de usuario. Solo se utiliza en el inicio de sesión.
+ Ejemplo: "{username}".
password string Este parámetro es opcional y define la contraseña del usuario. Solo se utiliza en el inicio de sesión.
+ Ejemplo: "{password}".
cURL
curl --user '{username}:{password}' -X 'POST' -H 'Content-Type: application/json' 'https://www.inteligente.com.ar/api/usuarios/sesion' -d '{"action":""}'
